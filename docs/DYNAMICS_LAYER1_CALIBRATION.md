# Validation Ladder · Layer 1 — калибровка линейки d_eff

*Слой 1 из валидационной лестницы спайн-документа («Канонар: ограниченно
рациональный агент и закон гейтированной размерности», §4). Цель слоя:
зафиксировать **один** эстиматор эффективной размерности `d_eff` и проверить
его на синтетике с известным ответом — **до** применения к Канонару. Без этого
любая оценка `d_eff(z)` нефальсифицируема (подгонка под результат).*

Код: `kanonar_behavior_lab/src/dynamics/` (Python, numpy). Это аналитический
слой, а не движок: TS-движок экспортирует траектории → CSV/JSON → этот эстиматор.

## Что зафиксировано (frozen estimator)

`estimators.py` — параметры эстиматора вынесены в module-level константы и
**не подбираются под целевую систему**. Две взаимодополняющие меры:

| мера | что | калибруется на |
|---|---|---|
| `participation_ratio` | ЛИНЕЙНая `d_eff = (Σλ)² / Σλ²` по спектру ковариации | гауссовы блобы известного ранга |
| `correlation_dimension` | НЕЛИНЕЙНая `d_eff` (Grassberger–Procaccia): наклон `log C(r)` по `log r` в scaling-региона, с Theiler-окном | Lorenz / Rössler / Hénon / logistic |

Frozen GP-параметры: `EMBEDDING_DIM=10`, `THEILER_WINDOW=10`,
`GP_MAX_POINTS=4000`, scaling-окно `GP_LO_PCTL=1`…`GP_HI_PCTL=15` (перцентили
распределения попарных расстояний). Окно `(1,15)` выбрано совместной
калибровкой по всей батарее (минимум суммарного `|D2 − known|`); малые радиусы
снижают saturation-bias, т.к. корреляционная размерность определена в пределе
`r→0`.

## Синтетическая батарея

`synthetic.py` — детерминированные (seeded) генераторы с известным ответом:
Lorenz (`D2≈2.06`), Rössler (`≈1.99`), Hénon (`≈1.22`), logistic `r=4` (`≈1.0`),
periodic (предельный цикл), AR(1) и white-noise (стохастика, контроль), плюс
`gaussian_blob(rank)` для калибровки PR.

## Как прогнать

```bash
python -m kanonar_behavior_lab.src.dynamics.calibrate
```

Печатает таблицу, пишет long-format `data/reports/dynamics_calibration.csv` и
`data/processed/dynamics_calibration.json`, и **возвращает ненулевой код**, если
хоть один hard-anchor вне допуска (т.е. скрипт = gate).

## Результат калибровки (8/8 hard anchors PASS)

| система | readout | value | known | band |
|---|---|---|---|---|
| gaussian rank 1/3/5 | PR | 1.00 / 3.00 / 5.00 | 1 / 3 / 5 | ±0.3…0.9 |
| lorenz | D2_native | 1.98 | 2.06 | [1.71, 2.41] |
| rossler | D2_native | 1.89 | 1.99 | [1.64, 2.34] |
| henon | D2_native | 1.17 | 1.22 | [0.92, 1.52] |
| logistic_r4 | D2_native | 0.85 | 1.00 | [0.75, 1.25] |
| white_noise | D2_native | 2.80 | — | ≥ 2.5 (не схлопывается) |

Линейная PR — практически точная. Нелинейная `D2` на потоках в пределах ~4–8 %
от учебных значений. Контроль (white-noise) не схлопывается в низкую размерность.

## Известные ограничения (честно, → следующий инкремент)

- **Delay-embedded scalar путь завышает.** `D2_embedded` (что реально получит
  Канонар со скалярных observable) для Lorenz даёт ~2.56, для шума ~7. Нужен
  подбор `dim` по false-nearest-neighbors, не фикс `dim=10`. Сейчас это
  *soft/report-only* readout, не hard-anchor.
- **Чистый предельный цикл ломает small-r GP.** periodic читается как `D2≈0`:
  период-50 рекурренции набивают массу на малых радиусах за Theiler-окном.
  Вывод: тип аттрактора нельзя классифицировать одной `D2` — нужен RQA /
  recurrence + знак показателя Ляпунова. Это **классификатор аттрактора**,
  второй компонент слоя 1, ещё не построен.

## Дальше по лестнице

  есть, без перетюнинга.
