# ЧАСТЬ I. ТЗ конечной версии

## 0. Назначение и базовые принципы

Детерминированная, полностью объяснимая («стеклянная») симуляция ограниченно
рациональных агентов в живом мире — персонажи, предметы, локации,
коммуникация, взаимодействие. Организующие механики: гейтированная размерность
и удержанное противоречие C(t). «Провал без злодеев» — воспроизводимая механика.

**Принцип 1. Объяснимый ИИ без машинного обучения.** В контуре решения нет
обученных аппроксиматоров: ни нейросетей, ни LLM, ни выученных эмбеддингов.
«Обучение» — в двух легитимных формах:
- **внутримировая адаптация** — только через именованные, инспектируемые
  переменные состояния (EMA-давления, гистерезис/сатурация целей, belief-атомы +
  surprise, trust/relations, медленный дрейф); каждая единица «выученного» —
  атом или поле с трассой;
- **оценивание параметров** — подбор интерпретируемых θ под данные (CRN +
  CMA-ES). **CMA-ES не участвует в runtime decision loop: он только подбирает
  интерпретируемые параметры до запуска профиля.** После фита система остаётся
  побайтово-объяснимой.

**Принцип 2. Коммуникация = обмен атомами; большая по конечной мощности,
маленькая по первому контракту.** Никаких LLM-API. Речевой акт — типизированный
атом {speaker, addressee, kind, topic→атом, stance, magnitude}, входящий в S0
адресата как событие → appraisal → ToM → решение. Невербальный канал —
obs:nonverbal атомы, пространственно гейтированные (слышимость/приватность).
Текст — **презентационный слой** (детерминированные шаблоны поверх трассы),
никогда не влияющий на симуляцию. Единственный допустимый способ роста —
**лестница версий** (§1.4): каждая версия = один новый проверяемый причинный
канал с twin-тестом; расширение без twin-теста запрещено, иначе трасса станет
большой, но не объясняющей.

**Принцип 3. C(t) в клетке.** В коде, тестах, леджере, лаб-доках — только
«tension mass / удержанное противоречие». «Сознание» — только в
интерпретационном слое (docs_conceptual, лора, роман) после kill-тестов.
Греп-правило: слово «сознание» вне conceptual-слоя = ошибка ревью.

**Ярус A. Core (движок живёт):**
- **A1** rollout ≥2 живых агентов × N тиков, локация+предмет, мир меняется.
  *Мин. PASS:* golden-run — тот же сид ⇒ побайтово идентичный отчёт (hash);

- **A2** каждое действие объяснимо + контрфактируемо.
  *Мин. PASS:* 100% выбранных действий с непустым usedAtomIds; twin-diff
  непуст и содержит **first divergence tick** с указанием разошедшегося атома.
- **A3** память реальна.
  *Мин. PASS:* wipe-memory twin расходится в решениях ≤5 тиков после wipe;
  затухание следа согласуется с предсказанным t_1/2 (из α EMA) в пределах ×2.
- **A4** предметы/локации причинны.
  *Мин. PASS:* предмет-абляция меняет меню (≥1 кандидат появился/исчез) И/ИЛИ
  сдвигает исход в предсказанный знак с |Δp| ≥ 0.10 на 32 сидах; аналогично
  локационная манипуляция.
- **A5** коммуникация v0 работает.
  *Мин. PASS:* сцена С1 — P(retreat/give | с угрозой) − P(без) ≥ 0.15 на 32
  сидах, знак пре-зарегистрирован.

**Ярус B. Validation:** B1 η²/scale-contract (личность до исходов);
B2 ножницы+гистерезис+крюк через руминацию в rollout; B3 d_eff(z)+C(z),

# ЧАСТЬ II. Фазовый план

## ФАЗА I — ядро: мир живёт (ярус A)

### Шаг I-0. Подготовительный технический шлюз (НЕ малый — честно средний; = spine WP-A + WP-B)

Per-seed экспорт + seed-aware статистика (freeze до пере-прогона); PAM v2
(challenge/defy) отдельным versioned freeze (решение автора 2026-07-05: ДА);

contract-тест побайтовой неизменности + freeze `docs/TENSION_FUNCTIONAL.md`.
**Результат:** цикл v4 закрыт честно; C(t) считается; ничего замороженного не тронуто.

### Шаг I-1. MVP-0: вертикальный срез — НЕПРИКОСНОВЕНЕН ПО ПРОСТОТЕ

Только живой цикл: **агент → действие → изменение мира → реакция другого
агента → trace → twin-diff.**

- Состав: одна сцена, 2 живых агента (goalLabDeciderPlugin), 1 локация с
  картой (существующие lib/world), 1 предмет v0 (ресурс-токен в facts:
  принадлежность + take/give/seize меняют меню), 5–20 тиков; Communication v0
  (сцена С1: threaten-атом через существующий communicativeIntent → событие в
  S0 адресата следующего тика).
- Раннер `runMvpRollout(seed)` → JSON (тик×агент: выбор, usedAtomIds, события);
  `runTwins(base, {removeObject | wipeMemory | injectSpeechAtom})` → diff +
  first divergence tick.
- **Запрещено в MVP-0:** d_eff, η², гейт, полноценная память-валидация, C(t)
  как обязательный выход, dialogue templates, сложные предметы, невербалка.
  Распухает — режем.
- **DoD:** A1, A2 полностью; A4/A5 прототипно (знаки наблюдены, пороги — в I-2).
- **Результат:** первая живая демка; раннер — фундамент всего дальнейшего.

### Шаг I-2. Мир формализуется (закрывает ярус A целиком)

- Предмет v1: типизированный контракт (facts-принадлежность; possibility-гейты
  И контекст-оси; transfer/seize в Game). Freeze предсказаний → A4 в полную силу.
- Локация v1: оси publicity/privacy/authority как факторы; перемещение как
  действие. Freeze → A4.
- Communication v1: невербальный канал (сцена С2), пространственные гейты;
  презентационный слой (шаблоны поверх трассы) отдельным модулем. → A5 закреплён.
- Память: пре-регистрация t_1/2, wipe-twin → A3.
**Результат: ярус A закрыт по минимальным PASS.**
